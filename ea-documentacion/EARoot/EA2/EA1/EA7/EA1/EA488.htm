Este CU tiene como objetivo la administración de Solicitudes de acciones entre áreas. Estas solicitudes pueden ser:
1. Anulaciones de períodos de deuda (Destino: Dpto. Emisión)
2. Retroactivos de períodos de deuda (Destino: Dpto. Emisión)
3. Condonaciones de períodos de deuda (Destino: Dpto. Emisión)
4. Liberaciones de períodos de deuda (Destino: Dpto. Emisión)
5. Verificar exencion de una cuenta (Destino: Dpto. Exención)
6. Asignación de broche a una cuenta (Destino: Dpto. Emisión)
7. Verificar cuenta para caso social (Destino: Dpto. Judiciales)
8. Modificación de datos (Destino: Dpto. Emisión)
Nota: En todos los casos, cada tipo de solicitud tiene asociada un área destino.
Las solicitudes podrán generarse de forma automática (caso: recepción de novedades de catastro) o con la intervención de un usuario.

Scenario Type Detail
Buscar Solicitud Alternate
Permite consultar las Solicitudes disponibles en el SIAT por:
. Area Origen
. Area Destino
. Fecha Desde
. Fecha Hasta
. Número
. Estado

Si existen resultados se muestra de cada uno de los registros: Fecha, Area Origen, Area Destino, Asunto, Estado y las acciones asociadas: "Ver Solicitud", "Modificar Solicitud", "Cambiar Estado", "Eliminar Solicitud"

Para agregar una nueva Solicitud, existe la opción "Agregar Solicitud".

Independientemente de los controles de seguridad habitualmente imlementados por SIAT, para este caso, existen las siguientes restricciones:

. Para Modificar una Solicitud: el usuario logueado deberá ser el mismo usuario que realizó el alta de la solicitud y además la solicitud deberá estar en estado Pendiente.

. Para Eliminar una Solicitud: Idem.

. Para Cambiar el estado de una solicitud: el área del usuario logueado coincida con el área destino de la solicitud.
Agregar Solicitud Alternate
Para crear una nueva solicitud, el sistema presentará un formulario como se detalla a continuación:

- Area Origen: Descripción (solo lectura) del área correspondiente al usuario logueado
- Usuario Alta: Descripción (solo lectura) del usuario logueado

- Fecha Solicitud
- Tipo Solicitud: El sistema presenta una lista de los Tipos de Solicitud existentes junto con la descripción del área destino correspondiente al Tipo. (concatenado en el combo)
- Asunto
- Descripción

- Cuenta: El sistema permite realizar una búsqueda de cuentas.


Cuando al usuario completa el formulario, el sistema recupera los datos ingresados e invoca al CU del siguiente modo:

. ID: lo genera automáticamente
. fecha Alta: fecha ingresada por el usuario
. Area Origen: área correspondiente al usuario logueado
. Usuario Alta: usuario logueado
. Tipo Solicitud: tipo ingresado por el usuario
. Area Destino: area correspondiente al Tipo Solicitud ingresado por el usuario.
. asunto: ingresado por el usuario
. descripción: ingresada por el usuario
. Cuenta: ingresada por el usuario (no requerido)
. Estado: Pendiente

. logSolicitud: "Creada - Pendiente - Fecha - Usuario"
Nota: logSolicitud es una observación donde se concatenan las acciones logueadas.


Luego invoca al CU: Alta Solicitud.
Eliminar Solicitud Alternate
El sistema visualiza un formulario con los datos de la solicitud y permite eliminarla.

Nota: Esta opción solo estará disponible siempre y cuando el usuario logueado sea el mismo usuario que realizaó el alta de la solicitud y la solicitud se encuetre en estado Pendiente.
Modificar Solicitud Alternate
El sistema visualiza el mismo formulario que en el caso del alta y permite modificarlo.

Nota: Esta opción solo estará disponible siempre y cuando el usuario logueado sea el mismo usuario que realizaó el alta de la solicitud y la solicitud se encuetre en estado Pendiente.
Cambiar estado Solicitud Alternate
El sistema presenta un formulario con los datos de la solicitud. (incluido el campo de log)

El sistema permite ingresar:

- Estado (Aceptado/Rechazado)
- Observaciones

El usuario debe seleccionar el estado e indicar las observaciones que crea convenientes.

El sistema actualiza le cambio de estado y agrega el registro de los a logSolicitud:
. logSolicitud: "Cambio Estado - Nuevo Estado - Fecha - Usuario"


Nota: Para poder realizarse un cambio de estado, el área destino de la solicitud debe coincidir con el área del usuario logueado.
