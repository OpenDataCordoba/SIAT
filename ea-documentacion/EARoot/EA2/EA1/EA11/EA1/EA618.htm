Administrar Corrida de Proceso Alternate
Recibe como parámetro de entrada un identificador de Corrida, el cual representa una instancia de un proceso a ejecutar con sus parámetros valorizados.

Una Corrida posee estados que indican, si el programa del proceso esta siendo ejecutado, si ya termino, o si esta por ejecutarse. También existen estados que indican la forma en que se ejecuto el programa del proceso.

Los estados posibles de una Corrida son:

- "En Preparación": permite el ingreso de la Fecha y Hora de ejecución del Proceso. Una vez ingresados dichos datos, se deberá "Activar" la corrida, cambiando el estado de la misma a "En Espera de Comenzar".

- "En Espera de Comenzar": representa una corrida que esta programada para ejecutarse, el comienzo puede ser Inmediato o a una determinada fecha y hora. Durante este estado, el usuario podrá Reprogramar la ejecución del proceso.


- "Procesando": indica que la corrida está siendo ejecutada.

- "Procesado con éxito": el programa termino con éxito y finalizó el proceso.

- "En espera continuar":el programa terminó con éxito, pero al finalizar indicó que todavía quedan pasos pendientes para finalizar el proceso, esperando la confirmación del usuario para continuar.

- "Procesado con error": el programa terminó con error.

- "Validación con error": el programa no pudo comenzar porque no pasó la validación.

- "Cancelado": el usuario canceló el proceso. ESta acción unicamente podrá ser ejecutada si el estado de la corrida es "En preparación", "En espera comenzar", "En espera continuar", "Procesado con error", y "Validación con error".

- "Abortado por excepción": representa una terminación anormal de la ejecución del programa en proceso.

Ademas una Corrida posee un Mensaje de Observación que se utiliza para brindar información a nivel usuario del Estado actual de la Corrida.
