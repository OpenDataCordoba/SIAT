Scenario Type Detail
Buscar Usuarios Alternate
Permite consultar los usuarios SIAT. Se puede buscar por usuarioSIAT.
Si existen resultados se muestra de cada uno de los registros el usuarioSIAT, y las acciones asociadas: "Visualización de Usuario", "Modificación de Usuario", y "Activar/Desactivar Usuario".
Para agregar una nuevo usuario, existe la opción "Agregar Usuario".
Visualizar Usuario Alternate
Permite visualizar información del usuario
Alta de Usuario Alternate
Para crear un nuevo de Usuario, se debe seleccionar:
. Nombre de usuario en SegWeb
. Area donde trabaja: El sistema muestra la lista de áreas
. Procurador: El sistema muestra una lista de los procuradores existentes más la opción: NO ES PROCURADOR.

El sistema deberá validar que el nombre de usuario ingresado exista en SWE. También deberá validar que no haya sido dado de alta en SIAT anteriormente.


Nota 1: En SIAT, existen tres tipos de usuarios: Usuario Interno: Son los usuarios que pertenecen a las áreas involucradas en el sistema. Los contribuyentes, que desde el púnto de vista del sistema son usuarios anónimos, y los Procuradores, que son usuarios que se conectan desde sus oficinas y que realizan ciertas acciones sobre SIAT. Existen otros casos como este último que a medida que se avance en el proyecto habrá que ir agregando. (investigadores, usuarios de registros de automotorores, etc.)

Nota 2: Durante el análisis, se identificaron casos donde es necesario contar con la lista de recursos con los que puede trabajar el usuario. Dicha funcionalidad será cubierta en releases posteriores.
Modificar Usuario Alternate
Permite visualizar los datos del usuario y modificar:

. Area donde trabaja: El sistema muestra la lista de áreas
. Procurador: El sistema muestra una lista de los procuradores existentes más la opción: NO ES PROCURADOR.

Nota: No se permite modificar el nombre de Usuario
Activar/Desactivar Usuario Alternate
Dependiendo del estado del Usuariose habilitará una de las dos opciones.
Si está en estado "Activo", se procederá a la desactivación del Usuario, pasando a estado Inactivo y no pudiendo ser referenciada nuevamente.
Si está en estado "Inactivo", se procederá a la activación del Usuario, pasando a estado Activo, pudiendo ser referenciada nuevamente.
