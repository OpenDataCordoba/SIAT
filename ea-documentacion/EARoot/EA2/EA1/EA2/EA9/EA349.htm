Este CU tiene como objetivo verificar si existen Exenciones asociadas a un Sujeto, cuando se recibe una novedad de Titular (alta/baja).

Scenario Type Detail
Novedades de Alta de Titular Alternate
1. Busco si el cuit de la persona recibida como parámetro está dentro de los sujetos exentos vigentes y si el recurso asociado a la exención del sujeto exento se relaciona con el tipo de objeto imponible recibido.
2. Si la búsqueda es verdadera
2.1 Verifico que las cuentas relacionadas con el objeto imponible recibido como parámetro no se encuentren ya exentas (en la tabla exe_cueExe)
2.2 Si no se encuentran
2.2.1 En la tabla de exenciones (exe_cueExe) doy de alta un registro para cada una de las cuentas que se relacione con el objeto imponible recibido como parámetro y con la fecha desde de exención la fecha de vigencia de la novedad.
Novedades de Baja de Titular Alternate
1 Busco todos los demás titulares del objeto imponible recibido como parámetro.
2 Si existe al menos uno
2.1 Por cada titular
2.1.1 Busco si el cuit del titular está dentro de los sujetos exentos vigentes y si el recurso asociado a la exención del sujeto exento se relaciona con el tipo de objeto imponible recibido
2.1.3 Si la búsqueda es falsa
2.1.3.1 Elimino todas las exenciones (que no tengan una solicitud relacionada) que existan para las cuentas relacionadas con el objeto imponible recibido como parámetro. Para eliminar, completo la fecha hasta de la exención con la fecha de vigencia de la novedad.

3 Si no existe otro titular
3.1 Elimino todas las exenciones (que no tengan una solicitud relacionada) que existan para las cuentas relacionadas con el objeto imponible recibido como parámetro. Para eliminar, completo la fecha hasta de la exención con la fecha de vigencia de la novedad.
