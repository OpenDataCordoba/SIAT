Este CU tiene como objetivo procesar novedades de Titulares de Cuentas provenientes desde otros Sistemas Externos a SIAT y actualizar los padrones correspondientes.

El CU recibe novedades de "Alta de Titular" y "Baja de titular" para una cuenta. Los sistemas externos considerados son Catastro, Habilitaciones, Cementerio, Serv. Públicos, Tribunal de Faltas y Rodados.

Procesar Novedades Titulares Cuenta Alternate
Los registros recibidos desde los Sistemas Externos presenta el siguiente formato:

1. codTipoObjetoImponible
2. codAreaOrigen
3. fechaAccion
4. accion
5. clave
6. fechaVigencia
7. idPersona
8. idTipoTitular
9. esPrincipal

Descripción:

1. codTipoObjetoImponible=PARCELA
2. codAreaOrigen=CATASTRO
3. fechaAccion= fecha en que ocurrió la acción en el sistema de catastro
4. accion=[ A | B | M ]
5. clave= es la clave con la que identificará unívocamente a cada registro de objeto imponible. En este caso, se piensa utilizar la CATASTRAL.
6. fechaVigencia= es la fecha de vigencia de la novedad.
7. idPersona= es id persona en la DB general.
8. idTipoTitular= es el id de tipo de tenencia en catastro
9. esPrincipal=indica si es titular principal (quien sale impreso en las boletas)

Notas:
- El idPersona debe existir en la DB general, caso contrario no se procede la actualización

- En caso de no existir como Contribuyente se deberá dar de alta y valorizar los atributos del contribuyente al valor por defecto
a estos fines. Luego se deberá dar de alta sobre titulares indicando en el caso que corresponda si es titular ppal.
Alta de un Titular Alternate
- Alta de un Titular (accion=A)

Ejemplo: | PARCELA | CATASTRO | 05/09/2007 | A | 0000001 | 05/08/2207 | 1354 | 1 | 1 |

Proceso:

- Obtener el Objeto Imponible a partir de la clave y Tipo de Objeto Imponible informados en la interface.
- Obtiener la lista de recursos que están relacionados con el Tipo de Objeto Imponible.
- Obtener el Recurso Principal
- Si no hay principal, fin procesamiento
- Si hay más de uno, informar error

- Verificar si "Modifica Titulares de Cuenta por Novedad de Objeto Imponible" en el recurso principal. Si no, fin de proceso.
- Obtener la Cuenta relacionada, en función del ID de Recurso principal.

- Validar que la persona informada exista como contribuyente.
- En caso que no exista realizar el alta
- En caso que exista obtener el ID de contribuyente. (que es igual al ID de persona)

- Generar el nuevo titular indicando: ID cuenta, ID contribuyente, fechaDesde=fechaActualización, tipoTitular y esPrincipal

// verificación de exenciones
- Obtener exenciones vigentes relacionadas con la cuenta
Si existe al menos una, generar una Solicitud de Verificar exencion de una cuenta por el alta de un nuevo titular.

// verificación de alta de sujeto exento
- Verificar si existe Sujeto Exento informado para el ID contribuyente
- Si existe, obtener ID exencion y ID broche
Si existe ID exencion, generar alta de exencion para cuenta
Si existe ID broche, asignar broche a cuenta (generando el histórico que corresponda0

- Para cada uno de los recursos secundarios, verifico si "Modifica Titulares de Cuenta por modicación de Titulares del Principal".
- Si modifica, obtengo la cuenta asociada al recurso secundario.
- Generar el nuevo titular indicando: ID cuenta, ID contribuyente, fechaDesde=fechaActualización, tipoTitular y esPrincipal.

- Verificar exencion
- Verificar alta de sujeto exento
Baja de un Titular Alternate
Baja de un Titular (accion=B)

Ejemplo: | PARCELA | CATASTRO | 05/09/2007 | B | 0000001 | 05/08/2207 | 1354 | 1 | 1 |

Proceso:

1. Obtener el Objeto Imponible a partir de la clave y Tipo de Objeto Imponible informados en la interface.
2. Obtiene la lista de recursos que están relacionados con el Tipo de Objeto Imponible.
3. Entre ellos, Busca el principal.
4. Si no hay principal, fin procesamiento
5. Si hay más de uno, informar error
6. Verifico si "Modifica Titulares de Cuenta por Novedad de Objeto Imponible" en el recurso principal. Si no, fin de proceso.
7. Obtener la Cuenta relacionada, en función del ID de Recurso principal.
8. Obtener el titular de la cuenta por el ID persona informado en la interface
9. Indicar fecha Hasta=fechaActualización
10. Se debe generar una solicitud de Verificar exencion de una cuenta por la baja de un titular.
11. Incluye el CU Validar Exención.
