Appears In: Objetos Imponibles
Este CU tiene como objetivo la administración del padrón de objetos imponibles.
En SIAT existirán padrones de: Parcelas, Comercios, Licencias, Sepulturas y cualuier otro Tipo de Objeto Imponible que pueda requerirse en el futuro.
Mediante el módulo de definición se permite identificar cuales son los atributos que posee cada Tipo de Objeto Imponible. Por ejemplo: para las Parcelas, se indicará que poseen: Radio, Zona, Superficie Cubierta, Valuación, etc.
Mediante este CU se permite realizar altas, bajas y modificaciones de los distintos objetos imponibles de cada tipo. Adicionalmente, se desarrollan las acciones que pudieran estar relacionadas con cada evento, de acuerdo a la configuración definida para cada Recurso. Por ejemplo, el alta de una Parcela, estará acopañada del alta de la cuenta de TGI relacionada y el alta de un Comercio estará acompañada del alta de la cuenta de DReI y Etur en caso que corresponda.

La modificación del valor de un atributo con vigencia retroactiva, genera una solicitud de análisis de retroactivo para el Dpto. Emisión. (lo mismo ocurre cuando se ingresa una revocación sobre una exención)

Scenario Type Detail
Buscar Objeto Imponible Alternate
Permite consultar los objetos imponibles disponibles en el SIAT mediante diferentes criterios de búsqueda. Permite buscar por Tipo de Objeto Imponible y Clave. (para el caso de Parcelas, la Clave será la Catastral, para el caso de Comercio, la Clave será el número de comercio)

Se permitirá realizar también una Busqueda Avanzada, donde se podrá buscar por los atributos definidos para cada tipo de objeto imponible, por rango o multivalor según corresponda. (siempre y cuando el atributo sea atributo de búsqueda).

Si existen resultados se muestra de cada uno de los registros clave, cuenta, fecha desde, fecha hasta, si está o no vigente y las acciones asociadas: "Ver Objeto Imponible", "Modificar Objeto Imponible", "Eliminar Objeto Imponible" y "Activar/Desactivar Objeto Imponible". Para agregar un nuevo objeto imponible al padrón, existe la opción "Agregar objeto imponible".
Ver Objeto Imponible Alternate
Permite visualizar información del Objeto Imponible seleccionado y los atributos del tipo de objeto imponible valorizados con sus vigencias.
Agregar Objeto Imponible Alternate
Para crear un nuevo objeto imponible inicialmente se debe indicar de que Tipo de Objeto Imponible se trata. Para ello el sistema presentará una lista de todos aquellos Tipos de Objetos que fueron indicados con el atributo: esSIAT. Una vez indicado el Tipo de Objeto imponible, el sistema mostrará todos los atributos definidos para el mismo, junto con una fecha desde.

Luego se deberán valorizar los atributos del Tipo de Objeto Imponible seleccionado.
Para dar de alta estos valores/atributos se deberá ingresar por cada uno uno el valor correspondiente.
Luego, con la Fecha de Alta y los valores para cada uno de los atributos, se prepara un Objeto Imponible para ser almacenado físicamente y se invoca al CU: "Alta Objeto Imponible".
Eliminar Objeto Imponible Alternate
Elimina en forma física el registro de objeto imponible si no está siendo referenciada. (Sólo se utilizaría para los casos que se acaba de dar de alta un objeto imponible y se descubre un error)
Modificar Objeto Imponible Alternate
Permite modificar la fecha de alta y cada uno de los atributos que componen al Objeto Imponible siempre y cuando estén marcados como: esAtributoSIAT.
Aquí el comportamiento es distinto dependiendo de si el atributo posee la marca de: poseeVigencia. (Los atributos con dicha marca son aquellos que podrían ocasionar un efecto retroactivo en la emisión del Recurso correspondiente)

Para los atributos que no poseen vigencia, al modificarse su valor, se reemplazará el valor actual por el nuevo.

Para los atributos que poseen vigencia la modificación implica la creación de un registro asociado a una nueva vigencia. En este caso, además del valor de altributo, se deberá informar: Fecha Novedad y Fecha Desde. La Fecha de Novedad representa el momento en que se conoce la Novedad (que generalmente debería ser el momento en que se carga) y la Fecha Desde representa la fecha desde la cual debería considerarse el efecto del cambio. Si la fecha Desde es anterior a la fecha actual, podría tratarse de un efecto retroactivo.

Luego, se prepara el Objeto Imponible con los cambios y se invoca al CU: "Modificar Objeto Imponible".
Activar/Desactivar Objeto Imponible Alternate
Se invoca al CU: Activar/Desactivar Objeto Imponible
